## Every Equation in BM3D — Explained with Intuition and Examples

I'll use a consistent running example throughout: our 8×8 image with a bright square (200) on a dark background (50), corrupted by Gaussian noise (σ = 25).

---

### Equation 0: The Observation Model

```
z(x) = y(x) + η(x),    x ∈ X
```

**What it says in plain English:**

```
noisy pixel = true pixel + random noise
```

- `z(x)` = the noisy image we observe (the only thing we have)
- `y(x)` = the true clean image (what we want to recover)
- `η(x)` = noise, drawn from a Gaussian distribution N(0, σ²)
- `x` = pixel coordinate (row, column)
- `X` = the set of all pixel coordinates in the image

**Example:**
```
At position (3,3):
  y(3,3)  = 200       (true: bright square)
  η(3,3)  = -8        (random noise)
  z(3,3)  = 200 + (-8) = 192   (what we see)
```

**Notation note:** `Z_xR` means a square block (patch) of size N₁ × N₁ extracted from the noisy image `z`, with its top-left corner at coordinate `xR`. The superscripts `ht` (hard-thresholding) and `wie` (Wiener) distinguish parameters used in Stage 1 vs Stage 2.

## STAGE 1: Basic Estimate

---

### Equation 1: The Ideal Distance (we can't compute this)

```
d_ideal(Z_xR, Z_x) = ||Y_xR - Y_x||² / (N₁ʰᵗ)²
```

**What it says:** If we had the clean image, we'd measure how different two patches are by:
1. Subtract one clean patch from the other, pixel by pixel
2. Square each difference
3. Sum them all up (that's the ℓ² norm squared)
4. Divide by the number of pixels (N₁ʰᵗ)² to normalize

**Example** with 3×3 patches (N₁ = 3):

```
Clean patch at xR (dark region):     Clean patch at x (also dark region):
┌──────────┐                         ┌──────────┐
│ 50 50 50 │                         │ 50 50 50 │
│ 50 50 50 │                         │ 50 50 50 │
│ 50 50 50 │                         │ 50 50 50 │
└──────────┘                         └──────────┘

Difference:  all zeros
||Y_xR - Y_x||² = 0² + 0² + ... = 0
d_ideal = 0 / 9 = 0          ← perfectly similar!
```

```
Clean patch at xR (dark region):     Clean patch at x (bright region):
┌──────────┐                         ┌────────────┐
│ 50 50 50 │                         │ 200 200 200│
│ 50 50 50 │                         │ 200 200 200│
│ 50 50 50 │                         │ 200 200 200│
└──────────┘                         └────────────┘

Difference: all 150
||Y_xR - Y_x||² = 150² × 9 = 202500
d_ideal = 202500 / 9 = 22500   ← very different!
```

**Problem:** We don't have the clean image `y`. We only have `z`. So this is a theoretical formula — it tells us what the "right" distance would be.

---

### Equation 2: The Noisy Distance (what we can actually compute)

```
d_noisy(Z_xR, Z_x) = ||Z_xR - Z_x||² / (N₁ʰᵗ)²
```

**What it says:** Same formula, but using the **noisy** patches instead of clean ones.

**Example:**

```
Noisy patch at xR:                   Noisy patch at x:
┌──────────┐                         ┌──────────┐
│ 38 72 55 │                         │ 61 44 68 │
│ 64 41 56 │                         │ 73 39 52 │
│ 45 62 43 │                         │ 39 56 48 │
└──────────┘                         └──────────┘

Both patches are from the dark region (true values all 50).
d_ideal should be 0 (they're identical clean patches).

But d_noisy:
  Differences: (38-61), (72-44), (55-68), (64-73), (41-39), (56-52), (45-39), (62-56), (43-48)
             =  -23,     28,      -13,    -9,      2,       4,       6,       6,      -5

  Sum of squares = 529 + 784 + 169 + 81 + 4 + 16 + 36 + 36 + 25 = 1680
  d_noisy = 1680 / 9 = 186.7
```

**The ideal distance was 0, but the noisy distance is 186.7!** The noise completely distorts the measurement.

---

### Equation 3: Why the Noisy Distance is Unreliable

```
E[d_noisy] = d_ideal + 2σ²

Var[d_noisy] = (8σ⁴)/(N₁ʰᵗ)² + (8σ² · d_ideal)/(N₁ʰᵗ)²
```

**What it says:**

- **The mean:** On average, the noisy distance equals the true distance **plus 2σ²**. So it's systematically shifted upward. Every pair of patches looks more different than it really is, by an amount 2σ².

- **The variance:** The noisy distance fluctuates randomly. The fluctuation grows with **σ⁴**. At high noise, the distances become so wildly random that you can't reliably tell which patches are similar and which aren't.

**Example (σ = 25):**
```
E[d_noisy] = d_ideal + 2 × 25² = d_ideal + 1250

For two identical clean patches (d_ideal = 0):
  E[d_noisy] = 1250     ← even identical patches look "1250 apart"!

For two different clean patches (d_ideal = 500):
  E[d_noisy] = 1750     ← truly different patches look "1750 apart"

The gap between 1250 and 1750 is small compared to the variance,
making it hard to distinguish similar from dissimilar patches.
```

**This is the problem that motivates the next equation.**

---

### Equation 4: The Smart Distance (with pre-filtering)

```
d(Z_xR, Z_x) = ||Υ(T₂Dʰᵗ(Z_xR)) - Υ(T₂Dʰᵗ(Z_x))||² / (N₁ʰᵗ)²
```

**What it says:** Before measuring distance, **pre-filter** each patch:
1. Apply a 2D transform `T₂Dʰᵗ` to each patch (e.g., DCT or wavelet)
2. Hard-threshold the coefficients with operator `Υ` (threshold = λ₂D · σ) — this removes the noisy small coefficients
3. Now compute the distance between these cleaned-up versions

**Example:**

```
Noisy patch at xR: [38, 72, 55, 64, 41, 56, 45, 62, 43]  (flattened 3×3)

Step 1: Apply 2D DCT transform
  DCT coefficients: [475.0, -23.1, 5.2, 30.4, -2.1, 7.8, -15.3, 1.9, -3.4]

Step 2: Hard threshold (λ₂D·σ ≈ 2.7 × 25 ≈ 68... but for smaller patches
         let's say threshold ≈ 20 for illustration)
  
  475.0  → |475.0| > 20 → KEEP → 475.0
  -23.1  → | 23.1| > 20 → KEEP → -23.1
    5.2  → |  5.2| < 20 → ZERO →   0
   30.4  → | 30.4| > 20 → KEEP →  30.4
   -2.1  → |  2.1| < 20 → ZERO →   0
    7.8  → |  7.8| < 20 → ZERO →   0
  -15.3  → | 15.3| < 20 → ZERO →   0
    1.9  → |  1.9| < 20 → ZERO →   0
   -3.4  → |  3.4| < 20 → ZERO →   0
  
  After: [475.0, -23.1, 0, 30.4, 0, 0, 0, 0, 0]

Do the same for the other patch, then compute distance between these
pre-filtered versions.
```

**Why this helps:** The small noisy coefficients that were confusing the distance measurement are **zeroed out**. Now the distance is computed only from the large, mostly-signal coefficients, making it much more reliable.

### Equation 5: The Matched Set

```
S_xR^ht = { x ∈ X : d(Z_xR, Z_x) ≤ τ_match^ht }
```

**What it says:** The set `S` contains all block positions `x` where the distance (from Eq. 4) is below a threshold `τ_match^ht`. These are the "similar" blocks.

**Example:**
```
Reference at xR = (0,0). Threshold τ = 2500.

  Patch at (0,0): d = 0       ≤ 2500  → ✓ IN  (always matches itself)
  Patch at (0,4): d = 180     ≤ 2500  → ✓ IN
  Patch at (5,5): d = 420     ≤ 2500  → ✓ IN
  Patch at (7,5): d = 1800    ≤ 2500  → ✓ IN
  Patch at (2,2): d = 18500   > 2500  → ✗ OUT  (bright region, too different)
  Patch at (3,4): d = 21000   > 2500  → ✗ OUT

S_xR^ht = {(0,0), (0,4), (5,5), (7,5)}     ← 4 matched blocks
```

Note: `d(Z_xR, Z_xR) = 0` always, so the reference block always matches itself. This guarantees the set has at least 1 element.

After finding this set, the matched blocks are stacked to form the 3D group `Z_{S_xR^ht}` of size N₁ × N₁ × |S_xR^ht|.

---

### Equation 6: Collaborative Hard Thresholding

```
Ŷ_{S_xR}^ht = T₃Dʰᵗ⁻¹ ( Υ( T₃Dʰᵗ( Z_{S_xR^ht} ) ) )
```

**What it says:** This is the core of Stage 1. Read it from inside out:

```
Step 1:  T₃Dʰᵗ( Z_{S_xR^ht} )     →  Apply 3D transform to the group
Step 2:  Υ( ... )                    →  Hard-threshold: zero out small coefficients  
Step 3:  T₃Dʰᵗ⁻¹( ... )            →  Inverse 3D transform: get back denoised patches
```

The hard-threshold operator `Υ` uses threshold `λ₃D · σ`.

**Example with our 4 matched patches (using 1 pixel position for clarity):**

```
Values at position [0,0] across 4 layers: [38, 61, 53, 42]
(All have true value 50)

Step 1: Apply 3D transform (Haar along stacking dimension)
  Coefficients: [48.5, 1.0, -11.5, 5.5]

Step 2: Hard threshold with λ₃D · σ = 2.7 × 25 = 67.5
  (For this simplified 1D example, let's use threshold = 15)
  
  48.5   → KEEP
   1.0   → ZERO
  -11.5  → ZERO
   5.5   → ZERO
  
  After: [48.5, 0, 0, 0]

Step 3: Inverse Haar transform
  Result: [48.5, 48.5, 48.5, 48.5]

Original:  38,   61,   53,   42     (errors: -12, +11, +3, -8)
Denoised:  48.5, 48.5, 48.5, 48.5  (errors: -1.5, -1.5, -1.5, -1.5)
```

The output `Ŷ` is a 3D array of denoised patches. Each individual 2D patch `Ŷ_x^{ht,xR}` is a "block-wise estimate" — meaning it's the estimate of the patch at position `x`, obtained through the group whose reference block was `xR`.

### Equations 10 and 11: Aggregation Weights

Before showing the aggregation formula, we need the weights.

#### Equation 10: Weight for Stage 1 (hard thresholding)

```
w_xR^ht = 1/(σ² · N_har^xR)     if N_har^xR ≥ 1
         = 1                      otherwise
```

Where `N_har^xR` = the number of non-zero coefficients that survived hard-thresholding.

**Intuition:** 
- If hard-thresholding killed almost everything (N_har is small), the representation was very sparse → very clean estimate → **high weight** (trust it more).
- If many coefficients survived (N_har is large), more noise leaked through → **low weight** (trust it less).

**Example:**
```
Group A: 3D transform had 64 coefficients. After thresholding, 4 survived.
  N_har = 4
  w = 1 / (25² × 4) = 1 / 2500 = 0.0004

Group B: 3D transform had 64 coefficients. After thresholding, 20 survived.
  N_har = 20
  w = 1 / (25² × 20) = 1 / 12500 = 0.00008

Group A gets weight 0.0004, Group B gets 0.00008.
Group A is 5× more trusted — because it was sparser.
```

#### Equation 11: Weight for Stage 2 (Wiener filtering)

```
w_xR^wie = 1 / (σ² · ||W_{S_xR^wie}||₂²)
```

Where `||W||₂²` is the sum of squares of all Wiener coefficients from Equation 8.

**Intuition:** Same logic. Wiener coefficients range from 0 to 1. If most are near 0, the sum of squares is small → aggressive filtering → trustworthy → **high weight**. If many are near 1, less filtering was done → less trusted → **low weight**.

**Example:**
```
Group A Wiener coefficients: [0.91, 0.04, 0.01, 0.02, 0.85, 0.03, ...]
  ||W||₂² = 0.91² + 0.04² + 0.01² + 0.02² + 0.85² + 0.03² + ...
           = 0.828 + 0.002 + 0.000 + 0.000 + 0.722 + 0.001 + ... ≈ 2.1
  w = 1 / (625 × 2.1) = 0.00076

Group B Wiener coefficients: [0.92, 0.88, 0.75, 0.81, 0.90, 0.79, ...]
  ||W||₂² ≈ 42.3    (many large coefficients → less filtering)
  w = 1 / (625 × 42.3) = 0.000038

Group A is ~20× more trusted than Group B.
```

### Equation 12: Aggregation (Weighted Average)

```
ŷ_basic(x) = [ Σ_{xR ∈ X}  Σ_{xm ∈ S_xR^ht}  w_xR^ht · Ŷ_xm^{ht,xR}(x) · χ_xm(x) ]
            / [ Σ_{xR ∈ X}  Σ_{xm ∈ S_xR^ht}  w_xR^ht · χ_xm(x) ]
```

This looks scary but is actually simple. Let me unpack it piece by piece.

**What `χ_xm(x)` means:** It's just a mask — equals 1 if pixel `x` falls inside the patch at position `xm`, and 0 otherwise. It's saying "only count this patch's contribution if it actually covers this pixel."

**What the formula says in plain English:**

```
For every pixel x:

  1. Look at ALL reference blocks xR that were processed
  2. For each xR, look at ALL patches xm in its matched group
  3. Each such patch gives an estimate for pixel x (if it covers x)
  4. Multiply each estimate by its group's weight w_xR
  5. Add them all up
  6. Divide by the sum of weights

  → It's just a weighted average of all estimates that cover pixel x.
```

**Example for pixel at position (3,3):**

```
Which patches cover pixel (3,3)?
  - Patch at xm=(2,2) from group with reference xR=(0,0): estimate = 197.0, weight = 0.0004
  - Patch at xm=(3,3) from group with reference xR=(1,1): estimate = 203.5, weight = 0.00008
  - Patch at xm=(2,2) from group with reference xR=(2,2): estimate = 198.2, weight = 0.0003
  - Patch at xm=(3,1) from group with reference xR=(3,3): estimate = 201.1, weight = 0.0002

Numerator   = 0.0004×197.0 + 0.00008×203.5 + 0.0003×198.2 + 0.0002×201.1
            = 0.0788 + 0.01628 + 0.05946 + 0.04022
            = 0.19476

Denominator = 0.0004 + 0.00008 + 0.0003 + 0.0002
            = 0.00098

ŷ_basic(3,3) = 0.19476 / 0.00098 = 198.7

True value = 200.  Error = only 1.3!
```

The key idea: **patches from sparser groups (lower N_har) contribute more to the final answer**, because they had more noise removed and are thus more reliable.

## STAGE 2: Final Estimate

Now we use `ŷ_basic` from Stage 1 to do everything better.

---

### Equation 7: Improved Block Matching (on the basic estimate)

```
S_xR^wie = { x ∈ X : ||Ŷ_xR^basic - Ŷ_x^basic||₂² / (N₁ʷⁱᵉ)² < τ_match^wie }
```

**What it says:** Same idea as Equation 5, but now we measure distance in the **basic estimate** instead of the noisy image. No pre-filtering needed because the basic estimate is already fairly clean.

**Example — why this is better:**

```
In the noisy image, two dark-region patches look like:
  Patch A: [38, 72, 55, 64, 41, 56, 45, 62, 43]
  Patch B: [61, 44, 68, 73, 39, 52, 39, 56, 48]
  Distance = big number     ← noisy, unreliable

In the basic estimate, the same patches look like:
  Patch A: [49, 51, 50, 48, 50, 51, 49, 50, 50]
  Patch B: [50, 48, 51, 50, 49, 50, 50, 51, 49]
  Distance = tiny number    ← clean, reliable!
```

Using these matched locations, **two groups** are formed:
- `Ŷ_{S_xR^wie}^basic` — stack of matched patches from the basic estimate (used to build the Wiener filter)
- `Z_{S_xR^wie}` — stack of patches from the **original noisy image** at the same locations (this is what gets filtered)

---

### Equation 8: Wiener Shrinkage Coefficients

```
W_{S_xR^wie} = |T₃D^wie(Ŷ_{S_xR^wie}^basic)|² / ( |T₃D^wie(Ŷ_{S_xR^wie}^basic)|² + σ² )
```

**What it says:** For each 3D transform coefficient, compute a weight between 0 and 1 using the classic Wiener formula:

```
W = signal_power / (signal_power + noise_power)
```

The "signal power" is estimated from the squared magnitude of the basic estimate's transform coefficients. The "noise power" is σ².

**Intuition:** This is the same logic as asking "what fraction of this coefficient is signal?"

**Example — computing W for individual coefficients:**

```
Basic estimate group → apply 3D transform → get coefficients

Coefficient #1: basic estimate gives magnitude = 120
  W₁ = 120² / (120² + 25²) = 14400 / 15025 = 0.958
  → "95.8% of this coefficient is signal" → keep almost all of it

Coefficient #2: basic estimate gives magnitude = 30
  W₂ = 30² / (30² + 25²) = 900 / 1525 = 0.590
  → "59% signal" → keep about half

Coefficient #3: basic estimate gives magnitude = 8
  W₃ = 8² / (8² + 25²) = 64 / 689 = 0.093
  → "9.3% signal" → suppress most of it

Coefficient #4: basic estimate gives magnitude = 2
  W₄ = 2² / (2² + 25²) = 4 / 629 = 0.006
  → "0.6% signal" → basically zero it out
```

**Compare to hard thresholding (Stage 1):**
```
                    Hard Threshold     Wiener Filter
Coeff = 120:        KEEP (100%)        KEEP (95.8%)      ← similar
Coeff = 30:         KEEP (100%)        KEEP (59.0%)      ← Wiener partially attenuates!
Coeff = 8:          KILL (0%)          KEEP (9.3%)       ← Wiener keeps a little bit
Coeff = 2:          KILL (0%)          KEEP (0.6%)       ← both essentially zero

The key difference is at medium-sized coefficients (like 30).
Hard threshold: all or nothing.
Wiener: proportional, gentle, nuanced.
```

### Equation 9: Collaborative Wiener Filtering

```
Ŷ_{S_xR}^wie = T₃D^wie⁻¹ ( W_{S_xR^wie} · T₃D^wie( Z_{S_xR^wie} ) )
```

**What it says:** Read from inside out:

```
Step 1:  T₃D^wie( Z_{S_xR^wie} )         →  3D transform the noisy group
Step 2:  W · (...)                         →  Multiply each coefficient by its Wiener weight
Step 3:  T₃D^wie⁻¹( ... )                →  Inverse transform → denoised patches
```

**Example (one pixel position, 4 layers):**

```
Noisy group values at position [0,0]:     [38, 61, 53, 42]
Basic estimate values at same position:   [48.5, 48.5, 48.5, 48.5]

Step 1: 3D transform of noisy group (Haar):
  Noisy coefficients: [48.5, 1.0, -11.5, 5.5]

Step 1b: 3D transform of basic estimate group (Haar):
  Basic coefficients: [48.5, 0, 0, 0]

Step 2: Compute Wiener weights (Eq. 8):
  W₁ = 48.5² / (48.5² + 25²) = 2352 / 2977 = 0.790
  W₂ = 0²    / (0² + 25²)    = 0    / 625   = 0.000
  W₃ = 0²    / (0² + 25²)    = 0    / 625   = 0.000
  W₄ = 0²    / (0² + 25²)    = 0    / 625   = 0.000

Step 3: Multiply noisy coefficients by Wiener weights:
  48.5 × 0.790 = 38.3
   1.0 × 0.000 =  0
  -11.5× 0.000 =  0
   5.5 × 0.000 =  0
  
  Filtered coefficients: [38.3, 0, 0, 0]

Step 4: Inverse Haar:
  All 4 patches get value: 38.3

Hmm, that's worse than Stage 1 (48.5). Why?
Because the basic estimate was [48.5, 48.5, 48.5, 48.5],
so the Wiener filter uses 48.5 as the "signal estimate."

In practice with real 2D patches (not single pixels), the basic estimate
is more accurate and the Wiener filter produces better results.
```

A more realistic example showing where Wiener outperforms hard-thresholding — **a coefficient of medium size**:

```
Suppose at some 3D transform position:
  Noisy coefficient  = 45
  Basic estimate coeff = 35

Hard thresholding (threshold = 67.5): 
  |45| < 67.5 → KILLED → 0     ← lost the signal!

Wiener filtering:
  W = 35² / (35² + 25²) = 1225/1850 = 0.662
  Result = 45 × 0.662 = 29.8     ← preserved most of the signal!

If the true coefficient was 30, then:
  Hard threshold error: |0 - 30| = 30      ← terrible
  Wiener filter error:  |29.8 - 30| = 0.2  ← excellent
```

**This is the fundamental advantage of Stage 2.**

### Equation 12 again: Final Aggregation

The final estimate `ŷ_final` uses the exact same aggregation formula as Equation 12, but with:
- `Ŷ_xm^{wie,xR}` instead of `Ŷ_xm^{ht,xR}` (Stage 2 estimates instead of Stage 1)
- `S_xR^wie` instead of `S_xR^ht` (Stage 2 matched sets)
- `w_xR^wie` instead of `w_xR^ht` (Stage 2 weights from Eq. 11 instead of Eq. 10)

## Summary: The Complete Algorithm Flow

```
EQUATION    WHAT IT DOES                           STAGE
────────    ──────────────────────────────────      ─────
Eq. 0       z = y + η (observation model)           setup

Eq. 1       d_ideal: true distance (can't compute)  ─┐
Eq. 2       d_noisy: noisy distance (unreliable)      ├── motivation
Eq. 3       Why d_noisy is unreliable                ─┘

Eq. 4       d: smart distance with pre-filtering     Stage 1
Eq. 5       S^ht: find matched blocks                Stage 1
Eq. 6       Collaborative hard thresholding           Stage 1
Eq. 10      Aggregation weights for Stage 1           Stage 1
Eq. 12      Weighted average → ŷ_basic               Stage 1

Eq. 7       S^wie: better matching on ŷ_basic        Stage 2
Eq. 8       Wiener shrinkage coefficients             Stage 2
Eq. 9       Collaborative Wiener filtering            Stage 2
Eq. 11      Aggregation weights for Stage 2           Stage 2
Eq. 12      Weighted average → ŷ_final               Stage 2
```

Each equation builds on the previous ones. Equations 1–3 explain why naive matching fails. Equation 4 fixes it. Equations 5–6 form the core of Stage 1. Equations 7–9 form the core of Stage 2. Equations 10–12 combine everything into the final image.
