## BM3D Explained with Examples

### Example Setup: A Tiny 8×8 Image

Imagine a very simple 8×8 grayscale image (pixel values 0–255, where 0=black, 255=white). It has a white square on a dark background:

```
True Image (y):                    
 50  50  50  50  50  50  50  50
 50  50  50  50  50  50  50  50
 50  50 200 200 200 200  50  50
 50  50 200 200 200 200  50  50
 50  50 200 200 200 200  50  50
 50  50 200 200 200 200  50  50
 50  50  50  50  50  50  50  50
 50  50  50  50  50  50  50  50
```

Now noise (σ=25) is added. Every pixel gets a random perturbation:

```
Noisy Image (z = y + noise):
 38  72  55  27  61  44  68  53
 64  41  56  48  73  39  52  47
 45  62 185 221 178 209  71  34
 58  36 214 192 225 187  44  63
 42  67 191 178 212 195  56  49
 55  38 209 223 188 211  40  58
 61  47  43  69  54  37  65  42
 39  56  48  52  46  61  38  57
```

The noise has corrupted every pixel. Some dark pixels (true value 50) now read as high as 73. Some bright pixels (true value 200) now read as low as 178. Our task: recover the clean image from this mess.

---

### Step 1: Block Matching — "Find Similar Patches"

Let's use **3×3 patches** for simplicity. Pick a reference patch from the top-left corner:

```
Reference Patch (at position [0,0]):
 38  72  55
 64  41  56
 45  62 185
```

Now we slide around the image looking for patches that are similar. We measure similarity by computing the **average squared difference** per pixel.

**Candidate 1** — patch at position [5,0] (bottom-left area):
```
 55  38 209
 61  47  43
 39  56  48
```

Distance = avg of (38-55)² + (72-38)² + (55-209)² + ... = **very large** (because 185 vs 209, 55 vs 209 differ a lot in structure). **Not similar. Rejected.**

**Candidate 2** — patch at position [0,4] (top-right area):
```
 61  44  68
 73  39  52
178 209  71
```

Distance = avg of (38-61)² + (72-44)² + (55-68)² + ... = **moderately small** (both patches look like "dark area with one bright corner"). **Similar! Accepted.**

**Candidate 3** — patch at position [5,5] (bottom-right area):
```
 37  65  42
 61  38  57
  ·   ·   ·
```

This is entirely in the dark region, while reference has a bright corner. **Not similar. Rejected.**

After scanning the whole image, suppose we find **4 similar patches** total (including the reference itself).

---

### Step 2: Grouping — "Stack Into a 3D Pile"

Stack the 4 matched patches into a 3D array:

```
Layer 1 (reference):    Layer 2:              Layer 3:            Layer 4:
 38  72  55              61  44  68            53  47  34          42  49  58
 64  41  56              73  39  52            63  44  63          67  56  40
 45  62 185             178 209  71            58 211  49          38  42 195
```

We now have a **3×3×4 cube** of data.

---

### Step 3: The 3D Transform and Hard Thresholding

This is where the magic happens. Let's focus on just **one pixel position** (bottom-right corner) across all 4 layers to see the 1D transform along the stacking axis:

```
Values at position [2,2] across all 4 layers:
185, 71, 49, 195
```

The true value at this position varies across patches — maybe some are in the bright region (true=200), some in the dark (true=50). But let's consider a simpler position.

**Position [0,0] across all 4 layers** (all are in the dark region, true value = 50 for all):

```
Noisy values: 38, 61, 53, 42
True values:  50, 50, 50, 50   (all the same!)
Noise:       -12, +11, +3, -8
```

Now apply the simplest 1D transform — a **Haar transform** (pairwise averages and differences):

```
Step 1: Pairwise
  Averages:    (38+61)/2 = 49.5    (53+42)/2 = 47.5
  Differences: (38-61)/2 = -11.5   (53-42)/2 = 5.5

Step 2: Next level  
  Average of averages:    (49.5+47.5)/2 = 48.5    ← DC (overall mean)
  Difference of averages: (49.5-47.5)/2 = 1.0     ← detail

Transform coefficients: [48.5, 1.0, -11.5, 5.5]
```

Now look at what happened:
- The **DC coefficient (48.5)** is close to the true value (50) — signal is concentrated here
- The **detail coefficients (1.0, -11.5, 5.5)** are small — they're mostly noise

**Hard thresholding** with threshold = λ·σ ≈ 3×25/√4 ≈ 15:
```
48.5  → |48.5| > 15 → KEEP   → 48.5
 1.0  → | 1.0| < 15 → KILL   →  0
-11.5 → |11.5| < 15 → KILL   →  0
 5.5  → | 5.5| < 15 → KILL   →  0

After thresholding: [48.5, 0, 0, 0]
```

**Inverse Haar transform:**
```
All 4 patches get the value: 48.5
```

Compare:
- Individual noisy values: 38, 61, 53, 42 (errors: -12, +11, +3, -8)
- After BM3D: 48.5, 48.5, 48.5, 48.5 (error: -1.5 for all)

**The error dropped from ±12 down to just 1.5.** That's the power of collaborative filtering.

### Now Consider a More Interesting Case

**Position [2,2]** — suppose 2 patches are from the bright square (true=200) and 2 from the dark background (true=50):

```
Noisy values: 185, 209, 49, 42
True values:  200, 200, 50, 50
```

Haar transform:
```
Coefficients: [121.25, 75.75, -12, 3.5]
```

Now the **second coefficient (75.75) is large** — it captures the real difference between bright and dark patches. Hard thresholding with threshold 15:
```
121.25 → KEEP
 75.75 → KEEP
-12    → KILL → 0
  3.5  → KILL → 0

After thresholding: [121.25, 75.75, 0, 0]
```

Inverse Haar:
```
Patch 1: 121.25 + 75.75 = 197.0
Patch 2: 121.25 + 75.75 = 197.0
Patch 3: 121.25 - 75.75 = 45.5
Patch 4: 121.25 - 75.75 = 45.5
```

Compare:
```
                     Noisy    BM3D    True    Noisy Error    BM3D Error
Patch 1 (bright):    185     197.0    200       -15            -3
Patch 2 (bright):    209     197.0    200        +9            -3
Patch 3 (dark):       49      45.5     50        -1           -4.5
Patch 4 (dark):       42      45.5     50        -8           -4.5
```

Even though the patches aren't identical, the transform was able to **separate the common pattern from the noise** and preserve the distinction between bright and dark patches.

---

### Step 4: Aggregation — "Combine Overlapping Estimates"

After processing every reference block, many patches overlap. Consider pixel at position [3,3] (true value = 200):

```
Estimate from group A (reference at [2,2]): 197.0
Estimate from group B (reference at [1,1]): 203.5
Estimate from group C (reference at [3,3]): 198.2
Estimate from group D (reference at [2,4]): 201.1
```

Each group also produced a **weight** (based on how many coefficients survived thresholding — fewer survivors = more aggressive denoising = more trusted):

```
Group A: weight = 0.8  (killed many noise coefficients → trustworthy)
Group B: weight = 0.3  (kept many coefficients → less trusted)
Group C: weight = 0.6
Group D: weight = 0.5
```

Weighted average:
```
Final estimate = (0.8×197.0 + 0.3×203.5 + 0.6×198.2 + 0.5×201.1) / (0.8+0.3+0.6+0.5)
               = 199.0
```

True value was 200. Error is only **1.0** — much better than the noisy value, which might have been 192 or 214.

### Stage 2: Why Do It Again?

#### The Problem with Stage 1

In Stage 1, we searched for similar patches in the **noisy** image. This causes two problems:

**Problem 1: Wrong matches.** Consider two patches:
```
Patch A (true: all 50):   38  72  55        Patch B (true: all 100):  88  112  95
                           64  41  56                                  114  91  106
                           45  62  43                                   95  112  93
```

After noise, Patch B's values (88-114) might look different enough from Patch A (38-72) to correctly reject. But at higher noise (σ=50), values could overlap heavily, and you might wrongly match patches that aren't truly similar.

**Problem 2: Crude thresholding.** Hard thresholding is a binary decision: keep or kill. A coefficient of value 14.9 gets killed while 15.1 gets kept. This is suboptimal.

#### How Stage 2 Fixes Both

**Fix 1: Match on the basic estimate.** After Stage 1, you have a cleaner image. Match patches in this cleaner image — far more accurate grouping.

```
Stage 1 basic estimate at dark region: 48.5, 49.2, 51.0, 47.8  (close to true 50)
vs.
Original noisy at same region:         38, 61, 53, 42  (scattered widely)
```

Matching on [48.5, 49.2, 51.0, 47.8] is obviously more reliable.

**Fix 2: Use Wiener filtering instead of hard thresholding.**

Example — for a given 3D transform coefficient:
```
Basic estimate's coefficient magnitude: 80  (clearly signal)
→ Wiener weight = 80² / (80² + 25²) = 6400/7025 = 0.91
→ Keep 91% of the noisy coefficient

Basic estimate's coefficient magnitude: 10  (probably noise)
→ Wiener weight = 10² / (10² + 25²) = 100/725 = 0.14
→ Keep only 14% of the noisy coefficient

Basic estimate's coefficient magnitude: 2  (almost certainly noise)
→ Wiener weight = 2² / (2² + 25²) = 4/629 = 0.006
→ Keep only 0.6% — essentially zero
```

Compare this to hard thresholding with threshold 15:
```
Coefficient = 80  → KEEP 100% (same as Wiener)
Coefficient = 10  → KILL 100% (Wiener keeps 14% — more nuanced!)
Coefficient = 2   → KILL 100% (same as Wiener)
```

The Wiener filter provides a **smooth, proportional** response instead of the abrupt binary decision. This preserves subtle details that hard thresholding would destroy.

### End-to-End Summary with Our Example

```
Original pixel (true = 200):                    200
After noise (σ = 25):                           185  (error = -15)
After Stage 1 (hard thresholding):              197  (error = -3)
After Stage 2 (Wiener filtering):               199  (error = -1)
```

Each stage reduces the error further:
- Noise introduced error of **15**
- Stage 1 reduced it to **3** (5× improvement)
- Stage 2 reduced it to **1** (another 3× improvement)

### Visual Analogy: Cleaning Up a Blurry Group Photo

1. **Block Matching** = You took 16 photos of the same scene, each with random camera shake. You identify which photos show roughly the same view.

2. **Stacking (Grouping)** = You align and stack those 16 photos on top of each other.

3. **Collaborative Filtering** = You look through the stack. Details that appear consistently in all 16 photos are *real*. Random blurs that appear in only 1-2 photos are *noise*. You keep the consistent details, discard the random ones.

4. **Two Stages** = First pass: rough alignment and cleanup. Second pass: now that the photos are cleaner, you can align them more precisely and do a more careful cleanup.

This is essentially what astrophotographers do when they "stack" hundreds of exposures of the night sky — and it's the same principle BM3D uses on a single photograph by finding repeated patterns within the image itself.
