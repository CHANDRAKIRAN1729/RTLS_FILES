## BM3D: Image Denoising by Sparse 3D Transform-Domain Collaborative Filtering

**Authors:** Kostadin Dabov, Alessandro Foi, Vladimir Katkovnik, Karen Egiazarian (Tampere University of Technology, Finland)
**Published:** IEEE Transactions on Image Processing, Vol. 16, No. 8, August 2007

---

### 1. Problem Setup

The observation model is simple:

```
z(x) = y(x) + η(x)
```

where `z` is the noisy image, `y` is the unknown true image, and `η` is i.i.d. zero-mean Gaussian noise with known variance σ². The goal is to estimate `y` from `z`.

---

### 2. Core Idea: Grouping + Collaborative Filtering

The paper's insight rests on **two pillars**:

#### 2.1 Grouping
- Take a 2D image block (a square patch of size N₁×N₁) as a **reference block**.
- Search through the image for other blocks that are **similar** to the reference block.
- **Stack** all these similar blocks into a **3D array** — this is called a **"group"**.
- Natural images are full of self-similarity (edges repeat, textures repeat, smooth areas repeat), so finding similar blocks is almost always possible.

#### 2.2 Collaborative Filtering
- Apply a **3D transform** to the group.
- **Shrink** (threshold or filter) the transform coefficients to attenuate noise.
- Apply the **inverse 3D transform** to get denoised estimates for every block in the group.

**Why does this work so well?** Within a group, there are two types of correlation:
- **Intra-fragment correlation**: pixels within each 2D block are correlated (natural image statistics).
- **Inter-fragment correlation**: corresponding pixels across different blocks are correlated (because the blocks are similar).

The 3D transform exploits **both** types of correlation simultaneously, achieving a **much sparser** representation than a 2D transform on each block alone. With `n` similar blocks each having `κ` significant 2D coefficients, independent 2D processing needs `n·κ` coefficients, whereas the 3D collaborative approach needs only about `κ` coefficients total — the sparsity enhancement grows with group size.

---

### 3. The Two-Step Algorithm

This is the heart of the paper. The algorithm has **two sequential stages**:

#### Step 1: Basic Estimate (Hard Thresholding)

**Step 1(a)i — Grouping by Block Matching:**
- For each reference block `Z_xR` in the noisy image, find all similar blocks within a local search window of size N_S × N_S.
- Similarity is measured via the ℓ²-distance. However, computing distance directly on noisy blocks is unreliable (variance grows as O(σ⁴)), so the paper applies a **pre-filtering**: transform each block with a 2D transform, hard-threshold the coefficients, then compute the distance on the thresholded spectra:

```
d(Z_xR, Z_x) = ||Υ(T_2D(Z_xR)) - Υ(T_2D(Z_x))||² / (N₁)²
```

- Blocks with distance < τ_match are considered similar and form the group.

**Step 1(a)ii — Collaborative Hard Thresholding:**
- Apply a **3D transform** (T_3D) to the group. This 3D transform is separable: a 2D transform (e.g., Bior1.5 wavelet) across each block, then a 1D transform (Haar wavelet) along the stacking dimension.
- **Hard-threshold** all 3D coefficients with threshold λ_3D·σ — coefficients below this are set to zero.
- Apply the **inverse 3D transform** to get denoised estimates of every block in the group.

**Step 1(b) — Aggregation:**
- Since blocks overlap and each block may appear in multiple groups, each pixel gets **many different estimates**.
- These are combined via **weighted averaging**, where the weight for a group is inversely proportional to its residual noise variance: `w = 1/(σ² · N_har)`, where N_har is the number of non-zero coefficients retained after hard-thresholding. Fewer surviving coefficients → more aggressive denoising → more trusted estimate → higher weight.

The output of Step 1 is `ŷ_basic`.

---

#### Step 2: Final Estimate (Wiener Filtering)

**Step 2(a)i — Improved Grouping:**
- Block matching is now done on the **basic estimate** `ŷ_basic` instead of the noisy image. Since the basic estimate has much less noise, the block distances are far more accurate — leading to **better groups**.
- Using the matched locations, **two groups** are formed: one from the noisy image `z` and one from the basic estimate `ŷ_basic`.

**Step 2(a)ii — Collaborative Wiener Filtering:**
- Apply the 3D transform to **both** groups.
- Compute **empirical Wiener shrinkage coefficients** from the basic estimate's spectrum:

```
W = |T_3D(Ŷ_basic)|² / (|T_3D(Ŷ_basic)|² + σ²)
```

This is the classic Wiener filter form: `signal_power / (signal_power + noise_power)`.

- **Multiply** the noisy group's 3D transform coefficients element-wise by these Wiener coefficients.
- Apply the **inverse 3D transform** to produce the final block estimates.

**Step 2(b) — Aggregation:**
- Same weighted averaging as Step 1, but now the weight is `w = 1/(σ² · ||W||₂²)`, where the squared ℓ²-norm of the Wiener coefficients measures the residual variance.

The output of Step 2 is the **final estimate** `ŷ_final`.

---

### 4. Why Two Steps?

The two motivations for Step 2 are:

1. **Better grouping**: Matching on the denoised basic estimate is far more reliable than matching on the noisy image, especially at high noise levels.
2. **Better shrinkage**: Wiener filtering using a pilot signal (the basic estimate) is a much more refined operator than hard-thresholding. Hard-thresholding either keeps or kills coefficients entirely; Wiener filtering *attenuates proportionally* based on estimated SNR per coefficient. This typically gains **>0.5 dB PSNR** over the basic estimate.

---

### 5. Efficient Implementation

The paper introduces several tricks to make the algorithm practical:

| Trick | Purpose |
|-------|---------|
| **Step size N_step** | Don't process every pixel as a reference — slide by N_step pixels (e.g., 3), reducing the number of reference blocks by ~N_step² |
| **Group size cap N₂** | Limit the max number of matched blocks (e.g., 16 or 32) |
| **Local search window N_S** | Don't search the whole image — restrict to a neighborhood (e.g., 39×39) |
| **Predictive-search BM** | Use previously matched locations to predict where to search next, forming smaller non-rectangular search regions |
| **Separable transforms** | Factor 3D transform into 2D (DCT or wavelet on blocks) + 1D (Haar across stacking dimension) |
| **Pre-computed 2D spectra** | Compute T_2D for each block once and reuse across overlapping search windows |
| **Kaiser window** | Apply to aggregation weights to reduce border artifacts from block transforms |

Overall complexity is **O(|X|)** — linear in image size.

---

### 6. Transform Selection

The paper systematically evaluates different transforms and finds:

- **The 2D transform choice barely matters** — even a "DC + random" orthonormal transform (one DC element, rest random) only costs ~0.1–0.4 dB. This means the inter-fragment correlation (exploited by T_1D along the stacking axis) is far more important than intra-fragment correlation.
- **The 1D transform needs a DC element** — DST (which lacks DC) performs worse. The "DC-only" case (pure averaging of blocks) performs very poorly (~2 dB loss), proving that non-DC basis elements are essential for capturing differences among grouped blocks.
- **Block ordering in the group doesn't matter** — confirmed by the fact that "DC+rand" for T_1D matches structured transforms.
- **Recommended transforms**: Bior1.5 wavelet for T_2D in Step 1, DCT for T_2D in Step 2, and Haar wavelet for T_1D.

---

### 7. Parameter Profiles

Two profiles are provided:

- **Normal Profile**: Higher quality, exhaustive-search BM. Further split into σ ≤ 40 (smaller blocks, no pre-filtering in distance) and σ > 40 (larger blocks, pre-filtering enabled to combat noisy block matching).
- **Fast Profile**: Uses predictive-search BM, ~6× faster with only 0.05–0.2 dB PSNR loss for moderate noise.

---

### 8. Color Extension (C-BM3D)

- Convert RGB to a luminance-chrominance space (e.g., opponent color transform).
- Perform block matching **only on the luminance channel** (Y), which has the highest SNR.
- **Reuse the same matched locations** for the chrominance channels (U, V) — this is the "grouping constraint."
- Collaborative filtering and aggregation are performed independently on each channel using the shared grouping.
- This yields 0.2–0.8 dB improvement over naively applying grayscale BM3D to each channel independently.

---

### 9. Key Takeaway

BM3D's power comes from the **synergy** of three ideas:
1. **Non-local self-similarity**: Natural images have many similar patches at different spatial locations.
2. **Sparsity enhancement through grouping**: Stacking similar patches into 3D arrays and transforming them produces a much sparser representation than 2D transforms alone.
3. **Two-stage refinement**: A coarse hard-thresholding step provides a pilot for a much more accurate Wiener filtering step — better grouping and better shrinkage compound to produce the final high-quality estimate.

At the time of publication, BM3D achieved state-of-the-art results on all standard test images, and it remains a benchmark reference in image denoising to this day.
