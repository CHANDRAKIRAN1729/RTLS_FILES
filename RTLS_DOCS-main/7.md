# Why GECO Instead of β-KL in VAE Training?

---

## The Standard β-VAE Approach

In a standard β-VAE, the loss function is:

$$\mathcal{L} = \text{MSE} + \beta \times \text{KL}$$

Where β is a **fixed hyperparameter** that balances reconstruction quality (MSE) against latent space regularization (KL).

---

## The Problem with Fixed β

### Problem 1: The Dilemma of Choosing β

| β Value | Effect on MSE | Effect on KL | Result |
|---------|---------------|--------------|--------|
| **Small** (β = 0.1) | Low MSE (good reconstruction) | High KL (poor regularization) | Overfitting, memorization, poor latent space |
| **Large** (β = 10) | High MSE (blurry reconstruction) | Low KL (good regularization) | Good latent space but unusable reconstructions |
| **Medium** (β = 1) | Maybe okay... | Maybe okay... | Depends on your specific dataset |

**There's no universal "correct" β.** It depends on:
- Dataset size
- Data dimensionality
- Network architecture
- The specific task

### Problem 2: β is Fixed Throughout Training

Even if you find a good β, it stays constant:

```
Early training:
  MSE = 10.0 (high, model just started)
  KL = 0.1 (low, encoder not learned yet)
  Loss = 10.0 + β × 0.1 = 10.0 + 0.1 = 10.1
  
  The model focuses on MSE (good!)

Late training:
  MSE = 0.001 (low, reconstruction is good)
  KL = 5.0 (high, encoder learned)
  Loss = 0.001 + β × 5.0 = 0.001 + 5.0 = 5.001
  
  Now KL dominates! Model may "forget" to reconstruct well.
```

The **optimal balance changes during training**, but β doesn't adapt.

### Problem 3: Gradient Scale Mismatch

The gradients from MSE and KL have different magnitudes:

```python
# Typical gradient magnitudes
∂MSE/∂θ ≈ 0.001   (small, because MSE is small after training)
∂KL/∂θ ≈ 1.0      (larger, KL doesn't shrink as much)

# With β = 1:
Total gradient = 0.001 + 1 × 1.0 = 1.001
# KL gradient dominates → model prioritizes KL over MSE
```

---

## GECO: The Solution

**GECO (Generalized ELBO with Constrained Optimization)** flips the problem:

Instead of:
> "Choose a fixed β to balance MSE and KL"

GECO asks:
> "What reconstruction quality (MSE) do I want? Let me automatically adjust λ to achieve it."

---

## How GECO Works: Detailed Explanation

### The Key Insight

GECO treats reconstruction quality as a **constraint** rather than a **competing objective**:

- **β-VAE**: "Minimize MSE + β×KL" (two competing objectives)
- **GECO**: "Minimize KL, **subject to** MSE ≤ τ" (one objective, one constraint)

### The Math

GECO solves:
$$\min_\theta \text{KL}(\theta) \quad \text{subject to} \quad \text{MSE}(\theta) \leq \tau$$

Using Lagrangian relaxation:
$$\mathcal{L} = \text{KL} + \lambda \times (\text{MSE} - \tau)$$

Which simplifies to:
$$\mathcal{L} = \text{KL} + \lambda \times \text{MSE} - \lambda \times \tau$$

Since λτ is constant w.r.t. θ, the effective loss is:
$$\mathcal{L}_{\text{effective}} = \text{KL} + \lambda \times \text{MSE}$$

**This looks like β-VAE!** But the key difference: **λ is dynamically updated**.

### The GECO Update Rule

```python
# At each training step:

# 1. Compute constraint violation
C = MSE - τ                  # How far from target MSE?

# 2. Smooth it (moving average)
CMA = α × CMA_old + (1-α) × C

# 3. Update λ based on constraint violation
κ = exp(step_size × CMA)     # Adjustment factor
λ = λ × κ                    # Update λ
λ = clamp(λ, λ_min, λ_max)   # Keep in bounds

# 4. Compute loss
Loss = KL + λ × MSE
```

### What the Update Does

| Situation | C = MSE - τ | CMA | κ = exp(step × CMA) | λ change |
|-----------|-------------|-----|---------------------|----------|
| MSE > τ (bad) | Positive | Positive | > 1 | λ increases |
| MSE < τ (good) | Negative | Negative | < 1 | λ decreases |
| MSE = τ (perfect) | 0 | 0 | 1 | λ stable |

## Why GECO is Better for This Task

### Reason 1: Robot Motion Planning Requires Precise Reconstruction

In robot motion planning, the decoder output must be **physically accurate**:
- Joint angles that actually achieve the end-effector position
- Small reconstruction errors can mean centimeters of workspace error

GECO lets you specify: "I need MSE ≤ 0.0008" and guarantees it.

### Reason 2: No Manual Tuning

| Approach | Tuning Required |
|----------|-----------------|
| β-VAE | Try β=0.1, train, evaluate. Too blurry? Try β=0.01. Still bad? Try β=0.001... |
| GECO | Set τ=0.0008 once. Done. GECO finds the right λ automatically. |

### Reason 3: Training Stability

β-VAE can have unstable training when β is wrong:
- β too small → model ignores KL → latent space collapses
- β too large → model ignores MSE → reconstructions useless

GECO maintains stability by continuously adapting λ.

### Reason 4: The CMA (Constraint Moving Average)

GECO uses a moving average to smooth the constraint signal:

```python
CMA = 0.95 × CMA_old + 0.05 × C_new
```

This prevents λ from oscillating wildly due to batch-to-batch noise.

|-------|-----|----|-----------------| -------------|

**Problem**: Once KL dominates the loss, the model has no incentive to improve MSE further.

| Epoch | MSE | KL | λ | Loss = KL + λ×MSE | What Happens |
|-------|-----|----|----|-------------------|--------------|
| 1 | 10.0 | 0.5 | 1 | 10.5 | MSE > τ, λ increases |

| 10000 | 0.0008 | 5.2 | 4471 | 8.8 | **MSE = τ, GOAL REACHED!** λ stabilizes |

**Success**: GECO pushed λ high enough to force MSE down to the target.

## The GECO Update Equation: Deep Dive

```python
# GECO update (from geco.py)

def loss(self, err, kld):
    # err = MSE, kld = KL divergence
    
    # Step 1: Constraint violation
    constraint = err - self.goal   # C = MSE - τ
    
    # Step 2: Exponential moving average
    if self.cma is None:
        self.cma = constraint
    else:
        self.cma = (1 - α) × constraint + α × self.cma
        # With α = 0.95:
        # cma = 0.05 × C_new + 0.95 × cma_old
    
    # Step 3: Compute adjustment factor
    with torch.no_grad():
        factor = exp(step_size × self.cma)
        # If cma > 0: factor > 1, λ increases
        # If cma < 0: factor < 1, λ decreases
        # If cma = 0: factor = 1, λ unchanged
    
    # Step 4: Update λ
    self.geco_lambda = (factor × self.geco_lambda).clamp(λ_min, λ_max)
    
    # Step 5: Return loss (NOTE: KL first, then λ × MSE)
    loss = kld + self.geco_lambda × err
    
    return loss
```

### Why Exponential?

The update is multiplicative: `λ_new = factor × λ_old`

Using `factor = exp(step_size × CMA)`:
- When CMA = 0: factor = exp(0) = 1, λ unchanged
- When CMA = 0.1: factor = exp(0.005 × 0.1) = 1.0005, λ grows 0.05%
- When CMA = 1.0: factor = exp(0.005 × 1.0) = 1.005, λ grows 0.5%
- When CMA = -0.1: factor = exp(-0.0005) = 0.9995, λ shrinks 0.05%

This gives **smooth, proportional** adjustments.

## GECO Parameters Explained

| Parameter | Symbol | Config | Your Value | Effect |
|-----------|--------|--------|------------|--------|
| **Goal** | τ | `g_goal` | 0.0008 | Target MSE. Lower = stricter reconstruction |
| **Step Size** | — | `g_lr` | 0.005 | How fast λ adapts. Higher = faster but may oscillate |
| **Smoothing** | α | `g_alpha` | 0.95 | CMA smoothing. Higher = slower, more stable |
| **Initial λ** | — | `g_init` | 1 | Starting weight. Doesn't matter much, GECO will adjust |
| **Min λ** | — | `g_min` | 0.00001 | Lower bound. Prevents λ → 0 |
| **Max λ** | — | `g_max` | 100000 | Upper bound. Prevents λ → ∞ |

---

## Final Summary: β-VAE vs GECO

| Aspect | β-VAE | GECO |
|--------|-------|------|
| **User specifies** | β (abstract weight) | τ (target MSE in interpretable units) |
| **Weight adjustment** | Fixed throughout training | Adaptive at every step |
| **Tuning** | Manual trial-and-error | Automatic |
| **Guaranteed reconstruction** | No | Yes (if τ is achievable) |
| **Training stability** | Depends on β choice | More stable (self-correcting) |
| **Loss formula** | MSE + β×KL | KL + λ×MSE (λ adaptive) |
| **Philosophy** | "Balance these two objectives" | "Achieve this constraint, then optimize that" |
