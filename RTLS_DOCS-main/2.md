# Simulation Scene Generation: Complete Validation Pipeline

The simulation script (new_simulate_in_moveit.py) generates and validates scenes using MoveIt as the ground truth collision checker.

---

## Constants

```python
MIN_EE_HEIGHT = 0.05        # EE must be ≥5cm above table
MIN_EE_OBSTACLE_DIST = 0.02 # EE must be ≥2cm from obstacle surface
MAX_SCENE_ATTEMPTS = 50     # Max retries to find valid scene
```

---

## The Validation Loop

```
while not valid_scene and attempts < 50:
    
    ┌─────────────────────────────────────────────────────────────────┐
    │ STEP 1: Sample q_start (random joint angles)                   │
    │ ─────────────────────────────────────────────────────────────── │
    │ q_start = rand(7) * (q_max - q_min) + q_min                    │
    │ e_start = FK(q_start)                                          │
    │                                                                 │
    │ CHECK: e_start.z ≥ 0.05m (above table)                         │
    │        ❌ FAIL → continue (resample)                            │
    └─────────────────────────────────────────────────────────────────┘
                                ↓ PASS
    ┌─────────────────────────────────────────────────────────────────┐
    │ STEP 2: Sample q_target (random joint angles)                  │
    │ ─────────────────────────────────────────────────────────────── │
    │ q_target = rand(7) * (q_max - q_min) + q_min                   │
    │ e_target = FK(q_target)                                        │
    │                                                                 │
    │ CHECK: e_target.z ≥ 0.05m (above table)                        │
    │        ❌ FAIL → continue (resample)                            │
    └─────────────────────────────────────────────────────────────────┘
                                ↓ PASS
    ┌─────────────────────────────────────────────────────────────────┐
    │ STEP 3: Check self-collision via MoveIt (no obstacles yet)     │
    │ ─────────────────────────────────────────────────────────────── │
    │ MoveIt scene: only table, no obstacles                         │
    │                                                                 │
    │ CHECK: collision_oracle.check_collision(q_start)               │
    │        ❌ FAIL (arm hits itself or table) → continue            │
    │                                                                 │
    │ CHECK: collision_oracle.check_collision(q_target)              │
    │        ❌ FAIL → continue                                       │
    └─────────────────────────────────────────────────────────────────┘
                                ↓ PASS
    ┌─────────────────────────────────────────────────────────────────┐
    │ STEP 4: Generate obstacle BETWEEN start and goal               │
    │ ─────────────────────────────────────────────────────────────── │
    │ t = rand(0.3, 0.7)  // 30-70% along the line                   │
    │ obs_pos = e_start + t * (e_target - e_start)                   │
    │ obs = [obs_x, obs_y, rand_height(0.5-1.0), rand_radius(0.05-0.15)] │
    │                                                                 │
    │ CHECK: dist_from_base = √(obs_x² + obs_y²) ≥ 0.25m             │
    │        ❌ FAIL (too close to robot base) → continue             │
    │                                                                 │
    │ CHECK: EE start not inside obstacle                            │
    │        dist_xy(e_start, obs) < obs_r + 0.02 AND e_start.z < obs_h │
    │        ❌ FAIL → continue                                       │
    │                                                                 │
    │ CHECK: EE target not inside obstacle                           │
    │        dist_xy(e_target, obs) < obs_r + 0.02 AND e_target.z < obs_h │
    │        ❌ FAIL → continue                                       │
    └─────────────────────────────────────────────────────────────────┘
                                ↓ PASS
    ┌─────────────────────────────────────────────────────────────────┐
    │ STEP 5: Check full-arm collision via MoveIt (with obstacles)   │
    │ ─────────────────────────────────────────────────────────────── │
    │ collision_oracle.add_obstacles(obstacles)                       │
    │                                                                 │
    │ CHECK: collision_oracle.check_collision(q_start)               │
    │        ❌ FAIL (robot ARM hits obstacle at start) → continue    │
    │                                                                 │
    │ CHECK: collision_oracle.check_collision(q_target)              │
    │        ❌ FAIL (robot ARM hits obstacle at target) → continue   │
    │                                                                 │
    │ collision_oracle.clear_obstacles()  // Reset for next attempt  │
    └─────────────────────────────────────────────────────────────────┘
                                ↓ PASS
    ┌─────────────────────────────────────────────────────────────────┐
    │ ✅ VALID SCENE! Exit loop                                       │
    └─────────────────────────────────────────────────────────────────┘
```

---

## What Each Check Ensures

| Step | Check | Why |
|------|-------|-----|
| 1 | `e_start.z ≥ 0.05m` | EE not below table |
| 2 | `e_target.z ≥ 0.05m` | EE not below table |
| 3 | MoveIt self-collision (no obs) | Robot doesn't hit itself or table |
| 4a | `dist_from_base ≥ 0.25m` | Obstacle is avoidable (not inside robot base) |
| 4b | EE not inside obstacle | Start/target are reachable |
| 5 | MoveIt full collision (with obs) | Robot ARM (all links) doesn't hit obstacle at start/target |

---

## Key Insight: Why Full-ARM Check?

Step 4b only checks if the **end-effector** is inside the obstacle. But the robot has multiple links:

```
                    ┌─────────┐
                    │ Obstacle│
                    │   ●     │
         ┌──────────┴─────────┴──────────┐
         │                               │
    Link 3    Link 4    Link 5    Link 6    EE ●
         │                               │
─────────┴───────────────────────────────┘
    
    EE is outside obstacle, but Link 4 collides!
```

Step 5 uses MoveIt's collision checking on the **full robot arm** to catch these cases.

```python
# Save for later use in evaluation
scenarios_to_save.append({

    'q_start': q_start.tolist(),      # [7] joint angles
    'e_start': e_start.tolist(),      # [3] xyz
    'q_target': q_target.tolist(),    # [7] joint angles (for reference)
    'e_target': e_target.tolist(),    # [3] xyz (the planning goal)
    'obstacles': [obs.tolist() for obs in obstacles]  # [[x,y,h,r], ...]
})
```

if loaded_scenarios:

**This ensures simulation and evaluation use identical scenes.**

## Summary: What's Guaranteed

| Property | Guaranteed by | Method |
|----------|---------------|--------|
| `q_start` within joint limits | Step 1 | Uniform sampling within limits |
| `e_start` above table | Step 1 | Height check (≥5cm) |
| `q_start` self-collision free | Step 3 | MoveIt check (no obstacles) |
| `q_start` arm doesn't hit obstacle | Step 5 | MoveIt check (with obstacles) |
| `q_target` same checks | Steps 2,3,5 | Same process |
| EE not inside obstacle | Step 4b | Geometric distance check |
| Obstacle not too close to base | Step 4a | Distance check (≥25cm) |
| **Feasible path exists** | ❌ NOT guaranteed | Not checked |
