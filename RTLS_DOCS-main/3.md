We discussed two cases above.

Ensuring validity of initial variables in simulation and then loading the same scenes in evaluation.
Ensuring validity of initial variables in evaluation itself.
Does these both do the same?

---

---

**No, they are NOT the same.** Here's the key difference:

---

## Comparison Table

| Validation Check | Simulation (MoveIt) | Evaluation (standalone) |
|------------------|---------------------|-------------------------|
| q_start within joint limits | ✅ Yes | ✅ Yes |
| e_start above table | ✅ Yes (check `z ≥ 0.05m`) | ❌ No |
| q_start self-collision free | ✅ Yes (MoveIt) | ❌ No |
| q_target same checks | ✅ Yes | ❌ No |
| Obstacle far from base | ✅ Yes (`≥ 0.25m`) | ✅ Yes (`≥ 0.30m`) |
| EE not inside obstacle | ✅ Yes (geometric) | ❌ No |
| Robot ARM collision at start/target | ✅ Yes (MoveIt) | ❌ No |

---

## Why the Difference?

**Simulation** has access to **MoveIt** (ROS-based collision checker):
- Can check full arm geometry
- Can verify self-collision
- Ground truth physics simulation

**Evaluation** (standalone) has **no MoveIt**:
- Only has the learned collision classifier
- Can only do simple geometric checks
- Runs faster, but less rigorous validation

---

## The Ideal Workflow

```
┌─────────────────────────────────────────────────────────────────┐
│ SIMULATION (with MoveIt)                                        │
│                                                                 │
│   Generate scene → Full validation → Save to JSON               │
│   (MoveIt checks)                                               │
└──────────────────────────┬──────────────────────────────────────┘
                           │
                           ↓ load scenes
┌─────────────────────────────────────────────────────────────────┐
│ EVALUATION (standalone)                                         │
│                                                                 │
│   Load pre-validated scenes → Plan → Measure success            │
│   (No additional validation needed - scenes already valid)      │
└─────────────────────────────────────────────────────────────────┘
```

---

## If Evaluation Generates Its Own Scenes

When evaluation generates scenes without loading:

```python
# Only these checks happen:
q_start = rand(7) * (q_max - q_min) + q_min  # ✅ Joint limits
e_start = FK(q_start)  # ✅ Valid EE

# Obstacle generation (ObstacleScenarioGenerator):
dist_from_base >= 0.3  # ✅ Obstacle not too close to base

# MISSING checks:
# ❌ EE height not checked
# ❌ Self-collision not checked  
# ❌ EE inside obstacle not checked
# ❌ Arm-obstacle collision not checked
```

**Risk**: Some generated scenes may have:
- Start/target below the table
- Robot in self-collision
- Robot arm intersecting obstacle at start/target

---

## Summary

| Approach | Quality | Speed |
|----------|---------|-------|
| Simulation → Save → Load in Evaluation | **High** (MoveIt validated) | Slower to generate |
| Evaluation generates own scenes | **Lower** (minimal validation) | Faster |

**Recommendation**: Always generate scenes in simulation (with MoveIt), save them, and load in evaluation to ensure fair comparison on identical, validated scenes.
